



# 项目名称 | 多语言翻译监修通知提醒与会话内确认功能

|------|-------------------------|
| 文档版本 | v2.0                    |
| 创建日期 | 2026年3月24日               |
| 修改记录 | v2.0 - 放弃自动触发方案，改为视觉强提醒方案 |

---

## 一、背景

### 1. 现状

当前游戏多语言翻译的协作链路中，日语翻译专家在完成对游戏日语的二次监修后，需要手动触发通知，告知其他语种的翻译专家进行后续监修。


| 角色     | 载体                     | 说明                      |
| ------ | ---------------------- | ----------------------- |
| 日语专家   | i18n text table（翻译工作台） | 完成游戏日语的二次监修工作           |
| 其他语种专家 | 消息中心                   | 接收日语专家发出的监修通知，并在完成后进行确认 |


**当前信息流转路径**：

```text
日语专家完成日语二次监修
    │
    ▼
日语专家手动点击 "Send Notification" 按钮  ⚠️ 容易遗漏
    │
    ▼
系统向其他语种专家发送通知
    │
    ▼
其他语种专家在【消息中心】收到通知  ⚠️ 不明显，易被忽略
    │
    ▼
其他语种专家完成监修并确认
```

### 2. 痛点

- **漏发通知导致流程阻塞**：日语专家在完成监修后，经常会忘记手动点击 `Send Notification` 按钮，导致下游其他语种的监修工作停滞，影响整体进度。
- **通知触达率低**：通知发送至其他语种专家的"消息中心"，该渠道信息繁杂且提醒不明显，专家容易漏看或延迟查看。
- **确认链路长**：其他语种专家缺乏便捷的确认入口，无法在沟通的上下文中直接完成工作交接。

### 3. 实现思路

> **为何不采用自动触发方案**：系统无法准确判断日语专家是否已完成监修——专家可能分多次编辑、中途暂停或仅修改部分词条，不存在明确的"监修完成"状态节点可供自动触发。

因此采用 **视觉强提醒 + 通知渠道优化** 的方案：

1. 在工具栏的通知按钮（喇叭图标）上，实时显示**红点徽标**，标注当前有多少词条的日语翻译已被修改但尚未通知其他语种专家，从视觉上持续提醒专家"还有未通知的变更"。
2. 将通知渠道从"消息中心"迁移至高频使用的 **i18n 咨询会话**中，通过 `@对应人员` 强提醒，并提供 `mark as reviewed` 快捷操作按钮。

**核心设计原则**：

- **视觉强提醒，降低遗忘率**：通过红点徽标 + 未通知变更数量，让"待发送通知"的状态始终可见，利用视觉压力驱动专家主动点击发送。
- **基于会话的协作 (ChatOps)**：在 i18n 咨询会话中通过 `@对应人员` 强提醒，并在会话消息中嵌入 `mark as reviewed` 按钮，实现"通知-处理-确认"的闭环。

---

## 二、目标

1、**降低漏发风险**：通过红点徽标的视觉强提醒，大幅减少专家忘记发送通知的情况。
2、**提升响应时效**：通过会话内 `@` 强提醒，提高其他语种专家的消息触达率和响应速度。
3、**优化协作体验**：提供会话内的 `mark as reviewed` 快捷操作，缩短专家确认路径，提升工作效率。

---

## 三、需求

### 1. 功能范围

- **提醒对象**：日语翻译专家（发送通知的操作人）。
- **提醒载体**：i18n text table 工具栏中的通知按钮（喇叭图标）。
- **通知目标**：对应的 i18n 咨询会话（i18n consultation session）。
- **通知形式**：系统在会话中发送包含 `@对应语种专家` 的提醒消息，并附带 `mark as reviewed` 交互按钮。

### 2. 功能需求清单

#### 2.1 喇叭按钮视觉强提醒（红点徽标 + 动效 + 弹窗提示）


| 需求项    | 说明                                                                                                                            |
| ------ | ----------------------------------------------------------------------------------------------------------------------------- |
| 触发条件   | 当本次操作中存在**日语（JA）翻译被修改**的词条时（无论涉及多少游戏、多少词条），触发视觉强提醒。                                                                           |
| 红点徽标   | 在喇叭图标右上角显示红色圆角徽标，标注**本次所有游戏中已修改日文的词条总数**（例如 `6`），实时更新。                                                                             |
| 图标动效   | 喇叭图标变为红色，并播放**持续的缩放 + 摇摆动画**（pulse），吸引专家注意力，直到专家发送通知后停止。                                                                      |
| 弹窗提示文案 | 点击喇叭图标打开 Send Notification 弹窗时，在弹窗顶部显示醒目的提示横幅：**"Japanese translations have been updated"**，提醒专家本次有日语变更需要通知。 |
| 数量统计规则 | 复用系统现有的 **Update 状态标记逻辑**。统计范围为当前表格中 `ja` 字段被标记为 Update（即存在 New/Old 变更）且尚未执行过 Send Notification 的词条总数。 |
| 实时更新   | 红点数量随编辑操作实时更新——新增修改时数字增加，发送通知后数字归零、红点隐藏、动画停止、提示横幅隐藏。                                                                          |
| 无变更时隐藏 | 当未通知数量为 0 时，红点徽标隐藏，喇叭图标恢复默认颜色和静态样式，弹窗中不显示提示横幅。                                                                                |


#### 2.2 Send Notification 弹窗


| 需求项   | 说明                                                                                           |
| ----- | -------------------------------------------------------------------------------------------- |
| 触发方式  | 点击工具栏中的喇叭图标，弹出 Send Notification 浮窗。                                                         |
| 弹窗内容  | 包含 Game 下拉选择器（单选）和 `Send Notification` 按钮。                                             |
| 发送反馈  | 点击 `Send Notification` 按钮后，无需二次确认，直接发送通知，并在页面顶部显示 Toast 提示："Notification sent successfully"。 |
| 发送后处理 | 发送成功后：1) **徽标数字扣除本次发送游戏的词条数量**（而非归零）；2) 对应词条的变更状态标记为"已通知"；3) 系统为本次发送的词条生成一个唯一的 **SubmissionID**，绑定到会话消息中下发；4) 在 i18n 咨询会话中发送通知消息。                          |
| 部分发送说明 | 若当前有多个游戏的 JA 词条已被修改，喇叭徽标显示的是所有游戏的合计数量。日语专家可选择其中某个游戏单独发送通知；发送后，徽标数字仅扣除该游戏对应的词条数量，剩余游戏的词条仍在徽标中显示，直到全部发送完毕才归零隐藏。 |


#### 2.3 会话内通知与强提醒 (@)


| 需求项    | 说明                                                                           |
| ------ | ---------------------------------------------------------------------------- |
| 消息触达渠道 | 在对应的 **i18n 咨询会话** 中发送通知消息。                                                  |
| @ 提醒机制 | 在发送的通知消息中，必须精准 `@` 到所有需要进行监修的其他语种专家，触发系统的强提醒机制。                              |
| 消息内容   | 明确告知日语监修已完成，提示被 `@` 的专家进行对应语种的监修工作。消息中应包含本次变更涉及的词条数量。                        |
| 消息中心跳转 | 保留原有"消息中心"的通知消息，但点击消息中心中的该通知后，不再展示原有详情页，而是**直接跳转到对应的 i18n 咨询会话**中，定位到相关通知消息。 |


#### 2.4 会话内快捷确认 (Mark as Reviewed)


| 需求项    | 说明                                                            |
| ------ | ------------------------------------------------------------- |
| 交互按钮   | 在通知消息下方/内部，提供一个 `mark as reviewed`（标记为已监修）的交互按钮。              |
| 权限控制   | 只有被 `@` 的对应语种专家点击该按钮才有效，非相关人员点击无效。                            |
| 状态同步   | 专家在会话中点击 `mark as reviewed` 后，系统需自动将该专家的监修状态同步回翻译管理系统，标记为已完成。 |
| 按钮状态变更 | 专家点击确认后，会话内的按钮状态需实时变更为"已确认/已监修"（或展示已确认人员名单），避免重复点击。           |

#### 2.5 会话入口与 SubmissionID 筛选

| 需求项 | 说明 |
| ----- | ----- |
| SubmissionID 生成 | 复用系统现有的 **Update 状态标记逻辑**来判断哪些词条需要被 check。日语专家点击 `Send Notification` 发送通知时，系统会自动筛选出当前被标记为 Update 状态的 JA 词条，为这些数据生成一个唯一的 SubmissionID，并将该 SubmissionID 与会话消息绑定后下发。 |
| 会话入口 | 其他语种专家（EN、ZH-TW、KO）收到会话中的 `@` 通知后，点击消息进入 i18n text table。 |
| 自动筛选 | 进入 i18n text table 时，系统根据消息中的 SubmissionID **自动筛选**出本次需要监修的词条：JA → EN、JA → ZH-TW、JA → KO 三条翻译方向的词条，并展示在表格中。 |
| 筛选范围 | 每个语种专家只看到与其负责方向相关的词条。例如 EN 专家只看到 JA → EN 的词条，不看到 JA → KO 的词条。 |
| 状态关联 | 专家在 i18n text table 中完成某方向的监修（点击 Mark as Reviewed）后，对应词条的该方向状态同步更新为"已完成"。 |

#### 2.6 退出提醒弹窗（Leave Warning）

| 需求项 | 说明 |
| ----- | ----- |
| 触发条件 | 日语翻译专家在 i18n text table 中修改了日语（JA）内容后，若在**未发送通知**的情况下尝试退出页面（点击标题栏左侧的返回按钮、左侧菜单栏任意项目、或浏览器关闭/刷新），系统弹出提醒弹窗。 |
| 弹窗文案（英文） | **"You have modified Japanese translations. Do you want to send notifications to translators?"** |
| 弹窗按钮 | 两个按钮：**"Exit"**（直接退出页面）和 **"Send"**（唤起 Send Notification 弹窗）。 |
| 行为 | 点击 "Exit"：直接关闭弹窗，正常退出/跳转，不发送通知。点击 "Send"：关闭弹窗，打开 Send Notification 浮窗，用户可正常选择游戏并执行通知发送流程。 |
| 不触发条件 | 若本次没有修改任何 JA 词条（即红点未出现），退出时**不弹出**该提醒。 |

## 四、附录

### 流程图（优化后）

```text
日语专家在 i18n text table 中修改日语翻译
    │
    ▼
系统实时统计"已修改但未通知"的词条数量
    │
    ▼
喇叭图标显示红点徽标（数字）+ 动画  ← 视觉强提醒
    │
    ▼
日语专家点击喇叭图标 → 弹出 Send Notification 弹窗
    │
    ▼
选择 Game → 点击 Send Notification
    │
    ├──▶ Toast 提示 "Notification sent successfully"
    │
    ├──▶ 红点徽标归零/隐藏、动画停止
    │
    ├──▶ 生成 SubmissionID，绑定本次词条
    │
    └──▶ 在 i18n 咨询会话中 @EN / ZH-TW / KO 专家发送通知
              │
              ▼
         其他语种专家点击会话消息
              │
              ▼
         进入 i18n text table，根据 SubmissionID 自动筛选
         该专家负责方向的词条（JA→EN / JA→ZH-TW / JA→KO）
              │
              ▼
         专家完成监修，点击 Mark as Reviewed
              │
              ▼
         该方向状态同步为已完成
```
