# 背景

D 系统要成为“人-AI Agent-人”控制链的基础设施，从决策层开始，拆解任务，分发任务，并管理人类节点使用合适的工具高效完成工作。


# 目标

1、从工作者自行制定工作任务，转为AI根据岗位职责分派任务，并进行评估和验收。

2、对于决策层，能够支持制定目标，拆解子任务，随时给出特定人的工作情况、特定目标的完成进展

用户视图

* 一线员工（客服、翻译官、运营……）
  * 接收新任务
    * 操作型任务——人工执行提供产出物
    * 审查型任务——审查 AI 工作结果
  * 任务引导（完成任务所需的系统，质量衡量标准）
  * 督促/结果验收
    * 监控任务执行过程（如客服，是否在 CS、CS Expert 上有工作行为）
    * 汇报每日工作量、工作结果
* 决策者
  * 接受新任务
    * 协同型任务——帮助 AI Agent 完成子任务拆解
    * 审查型任务——审核 AI/人的工作结果
  * 任务目标发布 Agent
  * 汇总查看各岗位人类员工工作情况
  * 汇总查看各岗位 AI Agent 工作情况
* 维护者
  * works——构建并维护工作流
  * tools——为 AI Agent 和人构建工具（定制接口&定制前端页面交互）
  * agents——任务管理、任务分析等专用 Agent
  * logs——AI 运行状态、token 消耗、计费
